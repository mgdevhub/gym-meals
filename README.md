## üß† Technical Depth: Gemini 3.0 Reasoning Engine

GymMeals is not just a wrapper. It is a **Reasoning Engine** built entirely around the **Gemini 3.0 Flash/Pro** API. We moved beyond simple image classification to "Multimodal Biological Analysis."

### üî¨ Anabolic Scoring Logic
The core of our engine is the **Anabolic Score (1-10)**. Gemini 3.0 doesn't just "see" food; it evaluates its biological value for hypertrophy based on:
1.  **Bioavailability**: How efficiently can the body utilize this protein?
2.  **Glycemic Stability**: Is this fuel compatible with the user's current metabolic state (e.g., Post-Workout vs. Bedtime)?
3.  **Leucine Threshold**: Does the meal trigger muscle protein synthesis?

### üëÅÔ∏è Vision Logic
We utilize Gemini 3.0's **Multimodal Reasoning** to understand *context*.
-   **Scenario**: A high-glycemic meal (Rice/Fruit).
-   **Logic**: If "Post-Workout" -> **Score: 9.0 (Optimal Fuel)**.
-   **Logic**: If "Rest Day" -> **Score: 4.0 (Insulin Spike Risk)**.
The engine adapts its "Verdict" instantly, offering "The Fix" (e.g., "Add fibers to blunt the insulin response") rather than just a calorie count.

## üõ°Ô∏è Iron Dome Security Protocol

Security is not an afterlife; it's our baseline. GymMeals implements the **Iron Dome** protocol to ensure user privacy in high-stakes environments.

## ‚öôÔ∏è Setup & Installation

1.  **Clone the Repository**
    ```bash
    git clone https://github.com/mgdevhub/gym-meals.git
    cd gym-meals
    ```

2.  **Configure Environment**
    Create a `.env` file in the root directory and add your **Gemini API Key**:
    ```bash
    EXPO_PUBLIC_GEMINI_API_KEY=your_api_key_here
    ```
    *(Get your key from [Google AI Studio](https://aistudio.google.com/app/apikey))*

3.  **Install Dependencies**
    ```bash
    npm install
    ```

3.  **Run the App**
    ```bash
    npx expo start
    ```
